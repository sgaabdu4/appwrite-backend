# Contributing to Appwrite Backend Skill

Thanks for contributing! Guidelines below.

## How to Contribute

### Reporting Issues

- **Bug reports**: Open issue. Clear desc, repro steps, expected vs actual
- **Feature requests**: Open issue describing new pattern/feature
- **Documentation improvements**: PRs for typos, clarifications, examples welcome

### Pull Requests

1. **Fork** repo
2. **Create branch** (`git checkout -b feature/add-new-pattern`)
3. **Make changes** per guidelines below
4. **Test** with Claude Code, Cursor, or other agent
5. **Submit PR** with clear desc

## Guidelines

### Documentation Style

- Clear, concise language
- Code examples in **Dart**, **Python**, **TypeScript** where applicable
- Follow existing formatting
- SKILL.md under 500 lines; detailed content → `references/`

### File Structure

- `SKILL.md` - Main skill file (overview, critical rules, quick ref)
- `references/` - Detailed topic docs
- `README.md` - GitHub-facing docs

### Commit Messages

- Present tense ("Add feature" not "Added feature")
- First line under 50 chars
- Reference issues (`Fixes #123`)

## Code of Conduct

Respectful, inclusive, constructive. All here to improve AI-assisted dev.
